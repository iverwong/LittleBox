## 关键架构决策

### 1. value 类型收敛为 `number`

原方案泛型 `T extends string | number`，收敛为 `number`：

- M5 年龄、M10 敏感度两个下游场景都是数值
- 泛型会让 `nodes[] / value / onValueChange` 的类型推断复杂化
- 业务层需要 string key 时自行映射（`["少", "中", "多"][value]`），无损
- `nodes: number[]` 只表达离散数值位置，不自动推导语义 label；默认兜底仍是 `String(nodes[0]) / String(value) / String(nodes[last])`
- 如果某个数值需要对应文案，业务层通过 `centerLabel={(value) => ...}`、`leftLabel`、`rightLabel` 注入；例如年龄可用 `centerLabel={(value) =>` ${value} 岁 · ${ageLabelMap[value]}`}`

### 2. `age → birth_date` 由后端转换（影响 M5 API 契约，不阻塞本计划）

<aside>
⚠️

**背景**：[M5 讨论骨架：账号前端联调与测试入口](https://www.notion.so/M5-48df3f5640cd41d7bec26654a0ec604e?pvs=21) §当前建议的接口与数据口径 §1 原约定「前端收 age → 提交前由前端转换为近似 birth_date」。评估后判断应改为**后端转换**。

**理由**：

- 前端设备时间可被用户篡改 / NTP 偏差 / 时区错乱，同一 "12 岁" 在不同设备产出不同 birth_date，下游统计会错
- 后端时钟 = Asia/Shanghai 权威源（项目指引 §5.9），单点计算可审计
- 换算策略（取当年 1 月 1 日 vs 年末 vs 其他）未来若调整，后端改一处即可

**建议契约**：

- `POST /children` 请求体从 `{ birth_date }` 改为 `{ age: number }`
- 后端接收后计算 `birth_date`（具体策略由 M4 后端定夺并写入 API doc）
- DB 列 `birth_date` 保留，作为内部表示

**本计划的影响**：无（`DiscreteSlider` 只返回 `number`）。

**M5 计划的影响**：需更新 M5 骨架「接口与数据口径 §1」并同步 M4 后端契约 —— 由 Iver 在 M5 计划起草时回写。

</aside>

### 3. 拖动 `onValueChange` 节流策略

拖动时 thumb 实时跟随手势（worklet 内操作 sharedValue），但 `onValueChange` **仅在吸附值变化时触发**（通过 sharedValue 存 `lastSnappedIndex` 对比），避免每帧调用导致上层 re-render。

### 4. 中央 label 位置：固定顶部几何中心（HTML 原型 v1.2 §8 锁定）

不跟随 thumb；切换时仅替换文本。好处：

- 视觉焦点稳定，避免 thumb 附近的视觉抖动
- 天然支持 `(value) => ReactNode` 动态描述词渲染
- M5 年龄档位之间描述词长度差异大（"幼儿" vs "即将成年"），固定中心位置避免 label 超出边界

### 5. `centerLabel` 槽位三态 API

`string | ReactNode | (value: number) => ReactNode`：

- `string`：作为当前值文本直接渲染（20px semibold）
- `ReactNode`：完全接管中央区域 UI
- `(value) => ReactNode`：根据当前值动态渲染（M5 / M10 主用）

`leftLabel` / `rightLabel` 只支持 `ReactNode`（不支持 function，因为边界值不随 value 变化）。

### 6. Hit area 扩展

节点可视直径 **14px 主色实心圆**（v1.2 · `primary[500]`，所有节点统一主色填充），通过透明 `Pressable` 扩展至 26×26（上下各 6px padding），满足触屏目标下限。

### 7. 拖动手势：translationX 增量方案（替代 absoluteX - trackLeft）

原设想中 `e.absoluteX - trackLeft` 存在坐标系错配：`Gesture.Pan()` 的 `e.absoluteX` 是**屏幕绝对坐标**，而 `View.onLayout` 回调给的 `nativeEvent.layout.x` 是**相对父容器**的坐标，两者相减得不到 track 内相对 X；若要拿到 track 的屏幕绝对位置需异步 `ref.measureInWindow`，首帧未就绪期间 thumb 会偏移；父容器若滚动或做 transform，假设进一步崩坏。

**改用 translationX 增量方案**：

- `thumbX: useSharedValue<number>` 存 thumb 在 track 内的**绝对像素**（0 = track 左边缘）
- `startX: useSharedValue<number>` 记录每次拖动起始时的 `thumbX`
- `onStart` → `startX.value = thumbX.value`
- `onUpdate` → `x = clamp(startX.value + e.translationX, 0, trackWidthSV.value)` → 写回 `thumbX.value = x`
- 全过程只需要 track 的**宽度**（`onLayout` 给的 `layout.width` 即可），不依赖任何绝对位置
- 对父容器滚动、transform、异步测量一律免疫

选择理由：无异步依赖 · 无坐标系错位 · Reanimated v4 worklet 内全部走 shared value 读写 · 上层无感知。

### 8. worklet 闭包处理：nodes 数组走 runOnJS，layout 量走 shared value

- `nodes` 是 JS 数组，worklet 直接闭包 `nodes[index]` 会在 `nodes` 变化时持有旧快照。方案：worklet 只向 JS 抛 `index`，通过 `runOnJS(notifyChange)(index)` 在 JS 线程读 **最新 nodes**：`notifyChange = useCallback((i) => onValueChange(nodes[i]), [nodes, onValueChange])`
- `trackWidth`、`nodeGap` 等 layout 派生量**必须**放入 `useSharedValue`（`trackWidthSV`），在 worklet 内用 `*.value` 访问；不得直接引用 JS 闭包里的普通变量（会被冻结为 worklet 生成时的初值，多数情况为 0）
- `nodeCount = nodes.length` 是原始 `number`，闭包捕获安全；但手势需 `useMemo` 依赖 `nodeCount`，保证 nodes 数量变化时手势重建

## 视觉规格冻结（内联自 HTML 原型 v1.2 §8）

本节是本计划的**唯一视觉事实源**。所有 Step 的验证只对照本节，不跨 Notion 页引用。

### Token 清单

- **轨道**：高 6px · 非激活 `theme.palette.neutral[200]` · 激活段（thumb 左侧）`theme.palette.primary[300]` · 圆角 3px
- **节点圆点**：直径 14px 主色实心圆 · 所有节点统一 `theme.palette.primary[500]`（激活与非激活不做色差，由轨道激活段 + thumb 位置区分当前值） · 外描边 2px `theme.palette.neutral[50]`（与背景融合，总视觉尺寸 18px）
- **Thumb**：直径 24px 主色实心圆 · 填充 `theme.palette.primary[500]` · 3px `theme.palette.surface.paper` 外环（RN 实现二选一：① thumb 外套一个 30×30 `surface.paper` 圆形 `View`；② 单一 `View` 用多层 `shadow` 叠加模拟——效果一致，不占 24px 内部尺寸）· 阴影 `theme.shadow.sm` · 按下态 `transform: [{ scale: 1.05 }]`
- **左右边界 label**：字号 12（`theme.typography.xs`）· 颜色 `theme.palette.neutral[500]` · 距轨道 12px
- **中央 label**：双行结构——上行 20px semibold `theme.palette.neutral[900]`（当前值文本）+ 下行 13px `theme.palette.neutral[500]`（描述词，可选）· 固定组件顶部几何中心 · 不跟随 thumb · 切换时仅替换文本 · 最小预留高度 44px（双行兜底；单行 `centerLabel` 也保留 44px 容器高度避免拉条行跳动）
- **容器**：padding `20px 24px`（上 20 / 右 24 / 下 20 / 左 24） · 圆角 `theme.radius.lg`（16px）· 底色 `theme.palette.surface.paper` · 阴影 `theme.shadow.sm`

### Disabled 规格

- **作用范围**：仅拉条本体（轨道行 = 轨道 + 节点 + thumb）；中央 label / 左右 label / 容器背景与阴影保持正常，**不置灰**
- 拉条行外层 `View` `opacity: 0.6`
- **轨道激活段与非激活段统一 `theme.palette.neutral[200]`**（视觉上"无激活"）
- 所有节点降为 `theme.palette.neutral[500]` 实心（外描边 2px `theme.palette.neutral[50]` 保留）
- Thumb 填充降为 `theme.palette.neutral[500]` 实心（3px `theme.palette.surface.paper` 外环保留 · 阴影保留）
- 节点 `Pressable` `disabled={true}` · `Gesture.Pan().enabled(false)`

### 内部工具函数（Step 1 使用）

在 `DiscreteSlider.tsx` 顶部（组件外）定义：

```tsx
/** 在 nodes 中查找距 value 最近的节点索引。平手时返回更小的索引。 */
function findNearestIndex(nodes: ReadonlyArray<number>, value: number): number {
	let best = 0
	let bestDiff = Math.abs(nodes[0] - value)
	for (let i = 1; i < nodes.length; i++) {
		const diff = Math.abs(nodes[i] - value)
		if (diff < bestDiff) {
			best = i
			bestDiff = diff
		}
	}
	return best
}
```

## 共享类型

```tsx
// mobile/components/ui/DiscreteSlider/DiscreteSlider.types.ts
import type { ReactNode } from 'react'
import type { StyleProp, ViewStyle } from 'react-native'

export interface DiscreteSliderProps {
	/** 节点值列表；严格递增；至少 2 档 */
	nodes: ReadonlyArray<number>
	/** 当前值；必须在 nodes 内，否则 fallback 到最接近的节点并在 __DEV__ 下 warn */
	value: number
	/** 吸附值变化时触发（不每帧） */
	onValueChange: (value: number) => void
	/** 整体灰显 + 不可交互 */
	disabled?: boolean
	/** 极小值 label；默认渲染 String(nodes[0]) */
	leftLabel?: ReactNode
	/** 极大值 label；默认渲染 String(nodes[last]) */
	rightLabel?: ReactNode
	/** 中央 label；支持 string / ReactNode / (value) => ReactNode 三态；默认渲染 String(value) */
	centerLabel?: string | ReactNode | ((value: number) => ReactNode)
	/** 三个 label 显示开关；默认 true；关闭时对应区域不占高度 */
	showLeftLabel?: boolean
	showRightLabel?: boolean
	showCenterLabel?: boolean
	style?: StyleProp<ViewStyle>
}
```

- [ ]  `notifyChange = useCallback((index) => onValueChange(nodes[index]), [nodes, onValueChange])` —— **worklet 中用 `runOnJS(notifyChange)(index)` 桥接**，让 JS 线程读最新 `nodes`，规避 worklet 对数组闭包的陈旧快照（§关键架构决策 §8）
- [ ]  `pan` 用 `useMemo` 包裹，依赖 `[disabled, nodeCount, notifyChange]`（`nodeCount = nodes.length`）：
    - `onStart` → `startX.value = thumbX.value`
    - `onUpdate` → `x = clamp(startX.value + e.translationX, 0, trackWidthSV.value)` → 写回 `thumbX.value`；算 `gap = trackWidthSV.value / (nodeCount - 1)`、`nextIndex = Math.round(x / gap)`；仅当 `nextIndex !== lastSnappedIndex.value` 时更新并 `runOnJS(notifyChange)(nextIndex)`
    - `onEnd` → `snappedX = lastSnappedIndex.value * gap` → `thumbX.value = withSpring(snappedX, { damping: 18, stiffness: 200 })`
- [ ]  **轨道容器**（包住 `ds-track` / 节点行的外层 `View`）挂 `onLayout`：读 `e.nativeEvent.layout.width` 写入 `trackWidthSV.value`；同时按 `activeIndex` 初始化 `thumbX.value`（避免首帧 thumb 在 0 位置闪烁）
- [ ]  **外部 `value` 变化同步**：`useEffect` 监听 `[activeIndex, nodeCount]`，若 `trackWidthSV.value > 0` 则 `thumbX.value = withSpring((activeIndex / (nodeCount - 1)) * trackWidthSV.value, ...)`；同步 `lastSnappedIndex.value = activeIndex`
- [ ]  worklet 内所有 layout 量**必须**走 `trackWidthSV.value`，严禁直接引用 JS 闭包里的普通变量（会被冻结为 worklet 生成时的初值 0）

**核心代码**：

```tsx
import { useCallback, useEffect, useMemo } from 'react'
import type { LayoutChangeEvent } from 'react-native'
import { Gesture, GestureDetector } from 'react-native-gesture-handler'
import Animated, {
	runOnJS,
	useAnimatedStyle,
	useSharedValue,
	withSpring,
} from 'react-native-reanimated'

const THUMB_RADIUS = 12 // thumb 直径 24 / 2

// —— shared values ——
const thumbX = useSharedValue(0)
const startX = useSharedValue(0)
const trackWidthSV = useSharedValue(0)
const lastSnappedIndex = useSharedValue(activeIndex)

const nodeCount = nodes.length

// —— JS 侧 notify（始终读最新 nodes） ——
const notifyChange = useCallback(
	(index: number) => onValueChange(nodes[index]),
	[nodes, onValueChange],
)

// —— 手势 ——
const pan = useMemo(
	() =>
		Gesture.Pan()
			.enabled(!disabled)
			.onStart(() => {
				'worklet'
				startX.value = thumbX.value
			})
			.onUpdate((e) => {
				'worklet'
				const width = trackWidthSV.value
				if (width <= 0 || nodeCount < 2) return
				const x = Math.max(0, Math.min(width, startX.value + e.translationX))
				thumbX.value = x
				const gap = width / (nodeCount - 1)
				const nextIndex = Math.round(x / gap)
				if (nextIndex !== lastSnappedIndex.value) {
					lastSnappedIndex.value = nextIndex
					runOnJS(notifyChange)(nextIndex)
				}
			})
			.onEnd(() => {
				'worklet'
				const width = trackWidthSV.value
				if (width <= 0 || nodeCount < 2) return
				const gap = width / (nodeCount - 1)
				const snappedX = lastSnappedIndex.value * gap
				thumbX.value = withSpring(snappedX, { damping: 18, stiffness: 200 })
			}),
	[disabled, nodeCount, notifyChange],
)

// —— track 宽度采集 + thumb 初始位置 ——
const onTrackLayout = useCallback(
	(e: LayoutChangeEvent) => {
		const w = e.nativeEvent.layout.width
		trackWidthSV.value = w
		if (nodeCount > 1) {
			thumbX.value = (activeIndex / (nodeCount - 1)) * w
			lastSnappedIndex.value = activeIndex
		}
	},
	[activeIndex, nodeCount],
)

// —— 外部 value 变化同步 thumb ——
useEffect(() => {
	const width = trackWidthSV.value
	if (width > 0 && nodeCount > 1) {
		thumbX.value = withSpring(
			(activeIndex / (nodeCount - 1)) * width,
			{ damping: 18, stiffness: 200 },
		)
		lastSnappedIndex.value = activeIndex
	}
}, [activeIndex, nodeCount])

// —— 动画样式 ——
const thumbStyle = useAnimatedStyle(() => ({
	transform: [{ translateX: thumbX.value - THUMB_RADIUS }],
}))
```
