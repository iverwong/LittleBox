## 决策点（已锁定）

- 集成测试用**独立库 `littlebox_integration`**，与 `littlebox_test` 物理隔离，真 commit + 测试间 TRUNCATE 清理。
- audit 入队名 bug 走 **TDD 红→绿**：先写复现红测，再修转绿。
- LLM 统一用 **FakeLLM**，在 `factory.py` 开注入缝（已批准改生产代码）。
- 真 Redis：backend 容器内连容器 Redis，独立 db index 隔离 dev。
- 分支 `test/m9.5-integration`；提交遵循 Conventional Commits，按阶段拆分。

## 环境与纪律

- 宿主机无 Python / uv，命令一律走 `docker compose exec api ...`，测试在 api 容器内跑。
- `audit_worker` 无 `--reload`，改 worker 代码后真机/容器验证前必 `docker compose restart audit_worker`（集成测试用进程内 burst worker，不依赖容器 worker；Step 25 真机依赖）。
- 隔离铁律（沿用 conftest）：禁 subprocess 连真库 / httpx 直连真 server / 裸 `redis.Redis()` / 自建 engine。**唯一豁免**：专供集成的 redis db index 允许 `flushdb`，须在集成 conftest 注释里写清边界。
- 新库 `littlebox_integration` 纳入 `_prod_db_row_count_guard`（13 表行数兜底，`LB_SKIP_PROD_GUARD=1` 跳过）。

## 关键设计约束（务必先读）

> 客户端断连**不取消段一**。段二（`_stream_generator`）收到 `CancelledError` / `ConnectionError` 即退出；段一（`_run_llm_pipeline`）作为独立 bg task 照常跑完 commit②（完整 `accumulated` 落库）并在 `finally` 释放 `chat:lock:{sid}`、`running_streams.pop`。
因此「强杀」的后端正确行为 = 段二停、段一独立收口。集成测试须经 `rr.register_chat_task` 句柄 await 段一完成后再断言落库 / 锁状态。
>
