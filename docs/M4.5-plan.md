### 工程风格基线（内联自指引 §4.3 前端部分）

- **Expo SDK 54**（RN 0.76+ / React 19）
- **路由**：expo-router（详见下节）
- **状态**：zustand + persist（业务）；auth / token 走 `expo-secure-store`；theme 走 React Context（MVP 静态）
- **类型**：TS strict；所有 props / hook 返值 / API 响应显式类型
- **命名**：组件 PascalCase，文件名同组件名；props / 返值走 `interface` / `type`
- **命令链**：`npm run lint` / `npm run typecheck` / `npm test` / `npx expo start`

### expo-router 路由规范（内联自指引 §5.7）

- **Route group `(name)` 仅用于顶层**：`(auth)` / `(child)` / `(parent)` 三类 persona / auth 分流；不产生 URL 段、不作为可路由子节点
- **Tab / Stack 子目标**：
    - 纯展示页 → 单文件 `children.tsx`
    - 需嵌套 Stack / 动态子路由 → `children/_layout.tsx` + `children/index.tsx` + `children/[id].tsx`
    - 文件夹形态必须加 `_layout.tsx`，否则 expo-router 注册为 `children/index` 而非 `children`
- **Auth-gate / 角色守卫**（按「角色与当前组匹配」判定）：

```tsx
if (role === 'parent' && !isParentGroup) router.replace('/(parent)/children')
if (role === 'child'  && !isChildGroup)  router.replace('/(child)')
if (role === null     && !isAuthGroup)   router.replace('/(auth)/login')
```

- **`unstable_settings.initialRouteName`**：值必须是当前 `_layout.tsx` 的直接子路由（Root 层为 `'(auth)'`），不能写嵌套路径
- **默认导航**：编程式 `router.push()`

## 关键架构决策（工程侧）

### 1. Theme 走 Context 而非 zustand

MVP 静态 theme，Context + `useMemo` 固化引用零订阅开销。

### 2. 样式工厂：每个组件的 `.styles.ts` 导出 `createStyles(theme: Theme)`

组件内 `const styles = useMemo(() => createStyles(theme), [theme])`。

### 3. 组件目录形态

每个组件 = 文件夹 + `ComponentName.tsx` + `.styles.ts` + `.types.ts` + `index.ts`。

### 4. 组件展厅（dev only 路由 `/(dev)/components`）

真机预览所有组件所有状态；M15 部署前负责移除条件构建（本计划不做）。

### 5. 语义色分层：UI 态色 vs 报告分类色

<aside>
⚠️

HTML 原型 v0.5 约定 `semantic` 四色（crisis / redline / guidance / safe）**仅用于家长端报告分类**。因此 token 拆两组：

- `ui`：`error` / `warning` / `success` / `info`（组件反馈，如 Input error / Toast warning）
- `report`：`crisis` / `redline` / `guidance` / `safe`（家长端日报图表专用）

两组颜色可以重用同一色值，但**语义 key 必须分离**，避免孩子端 UI 误用报告分类色。

</aside>

### 6. Toast 小型内部 store

Toast 队列用 `components/ui/Toast/toastStore.ts`（zustand 小切片），唯一在 `components/` 内出现的 store，因命令式 API 所需。

### 7. 吉祥物：**Lottie 方案**（非 SVG），占位先行外包到位内部替换

- **交付格式**：外包交付 **Lottie JSON**（AE + Bodymovin 导出）；集成库 `lottie-react-native`（非 `react-native-svg`）
- **本计划产出**：`<Mascot state="idle" size="md" />` 占位组件，内部 View + Feather icon 静态渲染；**API 完全对齐最终 Lottie 组件**（`MascotState` 6 态 + `onFinish` 一次性回调）
- **替换范围（最小化）**：
    1. 外包 13 个 JSON（6 state + 7 transition）放入 `mobile/assets/mascot/lottie/`
    2. `npm i lottie-react-native`
    3. 重写 `mobile/components/mascot/Mascot.tsx` 内部（Props 不变）
- 占位阶段**不引入图片资源**：占位视觉完全由 `Mascot.tsx` 的 View + Feather icon 渲染；替换时只动 `Mascot.tsx` 内部一文件
- 调用方 Props 契约不变；**状态机编排**（何时切 state）由 M5 / M7 负责，非本计划范围
- 完整外包交付契约见下文「§吉祥物集成基线」

### 8. 各核心组件视觉规格来源 = HTML 原型 v0.5

Button / Input / Card / ListItem / Toast / Avatar / Modal / Tag 的尺寸与视觉状态在 Step 2-5 逐个内联；数值严格以 HTML 原型 v0.5 的 CSS 为准。实施中发现偏差，先回 HTML 原型迭代（版本号 +1）再入代码。

### `mobile/components/mascot/Mascot.types.ts`

**关键**：本类型是占位组件与最终 Lottie 组件的**共同契约**。替换实现时不得修改本文件。

```tsx
import type { StyleProp, ViewStyle } from 'react-native'

// 对齐外包 Brief v1 的动效状态集
export type MascotState =
	| 'enter'      // 欢迎入场（一次性，1.2-1.8s）
	| 'idle'       // 静止呼吸（循环）
	| 'listen'     // 看向输入框（循环）
	| 'thinking'   // 思考中（循环，可无限延长）
	| 'narrating'  // 叙述中（循环，可无限延长）
	| 'done'       // 完成收束（一次性，0.6-1s）

export type MascotSize = 'sm' | 'md' | 'lg' | 'xl'   // 48 / 96 / 128 / 200 px

export interface MascotProps {
	/** 默认 'idle' */
	state?: MascotState
	/** 默认 'md' */
	size?: MascotSize
	/** 一次性动效（enter / done）结束回调；循环态不触发 */
	onFinish?: () => void
	style?: StyleProp<ViewStyle>
}
```

## 吉祥物集成基线（外包交付契约 · 内联自 Brief v1）

本节是本计划与外包的唯一契约。最终替换时按本节逐项对账，全部命中才合并。

### 动效状态清单（6 个）

| ID | 名称 | 循环 | 时长 | 触发场景 |
| --- | --- | --- | --- | --- |
| `enter` | 欢迎入场 | 一次性 | 1.2–1.8s | App 冷启动首次；末帧 = `idle` 首帧 |
| `idle` | 静止呼吸 | 循环 | 2–3s / 周期 | 默认待机；轻微呼吸 + 低频眨眼 |
| `listen` | 看向输入框 | 循环 | 2s / 周期 | 用户 focus / 输入中 |
| `thinking` | 思考中 | 循环·可无限延长 | 1.5s / 周期 | 用户发送后、LLM 首字到达前 |
| `narrating` | 叙述中 | 循环·可无限延长 | 1–1.5s / 周期 | LLM 流式输出正文时 |
| `done` | 完成收束 | 一次性 | 0.6–1s | LLM 输出结束；末帧 = `idle` 首帧 |

### 过渡片段（7 个，各 0.2–0.4s）

- `idle_to_listen` / `idle_to_thinking` / `listen_to_thinking`
- `thinking_to_narrating`（首字到达，头顶小灯泡一闪）
- `narrating_to_done`
- **`thinking_to_listen`** / **`narrating_to_listen`**（用户输入优先级 > 思考 / 叙述，必须能即时打断）

### 首末帧衔接规则

- 一次性动效（`enter` / `done`）末帧 = `idle` 首帧
- 所有循环动效首帧 = 末帧（完美闭环）
- `A_to_B` 片段：首帧 = A 某帧（README 注明），末帧 = B 首帧

### 技术约束

- **格式**：Lottie JSON（AE + Bodymovin），集成库 `lottie-react-native`
- **画布**：1024×1024（1:1）；实际渲染 48–200px；30 fps
- **背景**：透明，禁任何底色
- **素材**：纯矢量（禁位图 / 预渲染 PNG 序列）
- **体积**：主状态 ≤ 200KB；过渡 ≤ 80KB
- **音频**：全静音
- **表达式**：避开不被 Bodymovin 支持的 AE expressions；导出前在 `lottie-react-native` 实机测试

### 调色板（严格范围，高光 / 阴影仅叠加 `opacity`）

| 部位 | 色值（起 → 止） |
| --- | --- |
| 前板 | `#F0DFCB` → `#D2A882` |
| 右侧面板 | `#BE8B63` → `#A67148` |
| 盖板外侧 | `#E2C4A5` → `#BE8B63` |
| 盒内 | `#51331B` |
| 描边 | `#6D4627` |
| 眼珠 | `#3A2416` |

### 最终视觉方向（HTML 原型 v0.4 锁定）

- **单后翻盖子**（不是 2 片 V 字开口）；类似聊天头像里的迷你盒子
- **脸部仅 2 只简笔点目**（无嘴 / 无腮红 / 无眉）；情感通过眼睛 + 盒子姿态表达
- HTML 原型 v0.5 中三态 SVG 为 v0.3 placeholder，与最终方向**不同**，仅对照用

### 状态机切换参考（M5 / M7 调用方实现，非本计划范围）

- `enter` → `idle`（冷启动首次）
- `idle` ↔ `listen`（用户 focus / blur 输入框）
- `listen` → `thinking`（用户发送）
- `thinking` → `narrating`（首字到达）
- `narrating` → `done` → `idle`（LLM 结束）
- **打断**：用户在 `thinking` / `narrating` 期间重新输入 → 立即切 `listen`

// theme/ThemeProvider.tsx
import { createContext, useMemo, type ReactNode } from 'react'
import { theme as defaultTheme, type Theme } from './theme'

export const ThemeContext = createContext<Theme | null>(null)

export function ThemeProvider({ children }: { children: ReactNode }) {
	const value = useMemo(() => defaultTheme, [])
	return <ThemeContext.Provider value={value}>{children}</ThemeContext.Provider>
}

// theme/useTheme.ts
import { useContext } from 'react'

export function useTheme() {

	return ctx
}

**实现说明**：每个组件按「§文件结构」建 4 个文件。`.types.ts` 导出 Props 接口；`.styles.ts` 导出 `createStyles(theme)`；组件内用 `useMemo` 缓存。

```tsx
// Button.tsx 片段
export function Button({ variant = 'primary', size = 'md', loading, disabled, leftIcon, rightIcon, onPress, children, ...rest }: ButtonProps) {
	const theme = useTheme()
	const styles = useMemo(() => createStyles(theme), [theme])
	const isDisabled = disabled || loading
	return (
		<Pressable
			onPress={isDisabled ? undefined : onPress}
			style={({ pressed }) => [
				styles.base,
				styles[`variant_${variant}`],
				styles[`size_${size}`],
				pressed && !isDisabled && styles.pressed,
				isDisabled && styles.disabled,
			]}
			{...rest}
		>
			{/* leftIcon / children / rightIcon / loading 占位 */}
		</Pressable>
	)
}
```
