### Step B2：age ↔ birth_date 换算服务

`depends_on: none`

新增 `app/services/age_converter.py`。

**关键决策（Iver 2026-04-25 拍板）**：month/day 取**创建当日**（`today() − age` 年），后续可被 M10 家长精确生日覆盖。理由：相比固定 1 月 1 日 / 7 月 1 日，创建当日是用户实际出现时间，语义最清晰；「年龄 N 岁」在创建当日即是真值，跨日不会出现「明明 N 岁却被算成 N−1 岁」的边界跳档。

```python
def age_to_birth_date(age: int, ref: date | None = None) -> date:
	if not (3 <= age <= 21):
		raise ValueError(f"age must be in [3, 21], got {age}")
	ref = ref or date.today()
	try:
		return date(ref.year - age, ref.month, ref.day)
	except ValueError:
		# 闰年 2-29 兜底：平年用 2-28
		return date(ref.year - age, ref.month, ref.day - 1)

def birth_date_to_age(bd: date, ref: date | None = None) -> int:
	ref = ref or date.today()
	raw = ref.year - bd.year - ((ref.month, ref.day) < (bd.month, bd.day))
	return max(3, min(21, raw))
```

**TDD** `tests/test_age_converter.py`：

- 正向 + round-trip（注入 ref，age in 3..21）
- 越界抛 ValueError
- **闰年 2-29 兜底**：ref=2024-02-29, age=1 → 2023-02-28
- ref 注入避免依赖当前日期

**提交**：`feat(services): age↔birth_date converter (M4.8 B2)`

### Step B6：DELETE /children/{id} 级联硬删 + 审计

`depends_on: [B1, B4]`

CASCADE 链路 / 豁免项 / Redis 协同见 [技术架构讨论记录（持续更新）](https://www.notion.so/4ec9256acb9546a1ad197ee74fa75420?pvs=21) §八数据层「级联硬删范围」。本 Step 三块：DataDeletionRequest schema 重建 · FK CASCADE · 服务层写审计。

**0) DataDeletionRequest schema 重建**

当前 `status` 三态对「已完成硬删」无意义；`child_user_id` 带 FK → `users.id` 与 CASCADE 冲突。重建为：删 status 字段 + child_user_id 改 child_id_snapshot（无 FK，UUID 快照） + 加 deleted_tables JSONB。

```python
class DataDeletionRequest(BaseMixin, Base):
	"""数据删除请求审计（合规）。仅记录已完成的硬删。"""
	__tablename__ = "data_deletion_requests"

	requested_by: Mapped[uuid.UUID] = mapped_column(
		UUID(as_uuid=True),
		ForeignKey("users.id"),  # 保留 FK：parent 不会被删
		nullable=False,
		comment="发起删除的家长",
	)
	child_id_snapshot: Mapped[uuid.UUID] = mapped_column(
		UUID(as_uuid=True),  # 无 FK：child 已 CASCADE 删除，仅留 UUID 快照
		nullable=False,
		comment="被删 child 的 user.id（快照）",
	)
	deleted_tables: Mapped[dict] = mapped_column(
		JSONB,
		nullable=False,
		comment="{table: count} 各表删除行数",
	)
	reason: Mapped[str] = mapped_column(String(50), nullable=False)
	# id / created_at 由 BaseMixin 提供；created_at 即删除执行时间，不另加 executed_at
```

`app/models/enums.py` 删 `DeletionStatus` 枚举（重建后无引用）。

**1) FK 改 CASCADE**

`app/models/parent.py` Notification 加 child_user_id 列：

```python
child_user_id: Mapped[UUID | None] = mapped_column(
	ForeignKey("users.id", ondelete="CASCADE"), nullable=True
)
# payload JSON 注释：MVP 不约束 schema，消费方按 type 解构
```

一级 FK（→ `users.id`）全部加 `ondelete="CASCADE"`：`child_profiles.child_user_id` / `auth_tokens.user_id` / `device_tokens.user_id` / `family_members.user_id` / `sessions.child_user_id` / `daily_reports.child_user_id`

二级 FK（→ `sessions.id`）全部加 `ondelete="CASCADE"`：`messages.session_id` / `audit_records.session_id` / `rolling_summaries.session_id`

**2) 服务层 + handler**

新增 `app/services/child_deletion.py` 的 `hard_delete_child(session, *, child_user_id, requested_by)`：

1. **先 stage Redis auth 缓存清理**（CASCADE 后 auth_tokens 行消失，token_hash 取不回，必须 DELETE 前采集）
2. **CASCADE 前一次性 SELECT COUNT 各表行数**：child_profiles / sessions / messages / audit_records / rolling_summaries / daily_reports / notifications[child_user_id=?] / auth_tokens / device_tokens / family_members
3. `await session.execute(delete(User).where(User.id == child_user_id))` 触发 CASCADE
4. 写入 DataDeletionRequest（child_id_snapshot, deleted_tables, reason='parent_request'）

调用方必须用 `commit_with_redis(session, redis)` 提交。**禁用裸 `session.commit()`**——会留僵尸 Redis 缓存。

handler `app/api/children.py` DELETE：校 parent 身份 / 目标存在 / 同 family → 调服务 → 204。
