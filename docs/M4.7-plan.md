# M4.7 · 吉祥物动效 Native SVG 实现 — 实施计划 (4.7/17)

> 将 M4.5 落地的占位 Mascot 替换为基于 react-native-svg + Reanimated 4 的 6 态动效实现。设计决策见 [技术架构讨论记录（持续更新）](https://www.notion.so/4ec9256acb9546a1ad197ee74fa75420?pvs=21) §十二；6 态动效参数对齐结论已沉淀至本计划 §三 STATE_PARAMS / LAYOUT。
> 

---

## 一、目标概述

### 1.1 做什么

- 6 态动效（idle / listen / thinking / narrating / enter / done），每态动作复杂度按「中等基线」（核心动作 + 必要辅助动作）
- 4 尺寸（sm / md / lg / xl）等比缩放（viewBox 200×200 基准，动画参数单份）
- 一次性动效（enter / done）完成后触发 `onFinish`
- Mascot 对外 API（`MascotState` / `MascotSize` / `MascotProps` / `ONE_TIME_STATES`）保持 M4.5 契约，零改动

### 1.2 不做什么

- 不做 Skia 兜底（v2 §9 触发条件未达；M7 聊天页接入时再验证 FlatList / 键盘指标）
- 不做 ClipPath 盒口裁切（v2 §3 提案在本计划弃用，理由见 §五·5.3）
- 不做盒盖独立动效（盖子跟随 body 整体变换；后续动作专题升级）
- 不做 idle 随机视线游移（idle 仅做眨眼）
- 不做 narrating 多元素抛洒（首版仅单光斑）
- 不做单元测试（前端纯视觉组件，按指引豁免，仅展厅手动验证）
- 不做动效参数最终值锁定（首版按 §三 STATE_PARAMS / LAYOUT 落地，6 态动作专题确认后回写）

```tsx
import Svg, { Path, G, Ellipse } from 'react-native-svg'
import Animated from 'react-native-reanimated'
import {
	EYE_OPEN_D, EYE_SQUINT_D, EYE_SMILE_D, EYE_CRESCENT_D,
} from './animations'
import { THINKING_LAYOUT, NARRATING_LAYOUT } from './icons'
import { colors } from '@/theme/colors'

const AnimatedG = Animated.createAnimatedComponent(G)
const AnimatedPath = Animated.createAnimatedComponent(Path)
const AnimatedEllipse = Animated.createAnimatedComponent(Ellipse)

export interface MascotSvgProps {
	bodyAnimatedProps: any
	shadowAnimatedProps: any
	blinkAnimatedProps: any
	eyesOpenAnimatedProps: any
	eyesSquintAnimatedProps: any
	eyesSmileAnimatedProps: any
	eyesCrescentAnimatedProps: any
	thinkingAnimatedProps: any[]
	narratingAnimatedProps: any[]
}

export function MascotSvg(props: MascotSvgProps) {
	return (
		<Svg width="100%" height="100%" viewBox="0 0 200 200" style= overflow: 'visible' >
			{/* 阴影椭圆（最底层；enter 时由 shadowAnimatedProps 控制 scale + opacity） */}
			{/* 用 G wrapper 让 scale 中心固定在阴影中心 (100, 195)，避免 scale 时位移 */}
			<AnimatedG animatedProps={props.shadowAnimatedProps}>
				<Ellipse cx={0} cy={0} rx={40} ry={8} fill={colors.mascot?.shadow ?? '#000000'} />
			</AnimatedG>

			{/* 盒子整体（box-inner / box-body / box-lid / 脸 / 眼睛） */}
			<AnimatedG animatedProps={props.bodyAnimatedProps}>
				<Path d="..." fill="..." /> {/* box-inner from mascot.svg */}
				<Path d="..." fill="..." /> {/* box-body from mascot.svg */}
				<Path d="..." fill="..." /> {/* box-lid from mascot.svg */}
				<Path d="..." fill="..." /> {/* 脸 from mascot.svg */}

				{/* 眼睛 4 形态层叠，由 opacity 切换；外层 G 受 blink scaleY 控制 */}
				<AnimatedG animatedProps={props.blinkAnimatedProps}>
					<AnimatedPath animatedProps={props.eyesOpenAnimatedProps}     d={EYE_OPEN_D}     fill={colors.mascot?.eye ?? '#2A1F14'} />
					<AnimatedPath animatedProps={props.eyesSquintAnimatedProps}   d={EYE_SQUINT_D}   stroke={colors.mascot?.eye ?? '#2A1F14'} strokeWidth={3} strokeLinecap="round" fill="none" />
					<AnimatedPath animatedProps={props.eyesSmileAnimatedProps}    d={EYE_SMILE_D}    stroke={colors.mascot?.eye ?? '#2A1F14'} strokeWidth={3} strokeLinecap="round" fill="none" />
					<AnimatedPath animatedProps={props.eyesCrescentAnimatedProps} d={EYE_CRESCENT_D} stroke={colors.mascot?.eye ?? '#2A1F14'} strokeWidth={3} strokeLinecap="round" fill="none" />
				</AnimatedG>
			</AnimatedG>

			{/* thinking 抛物线抛出（每元素外层 G 控位置/旋转/缩放，内层 path 加 translate(-12 -12) 让 Lucide 24×24 中心对齐 G 原点） */}
			{THINKING_LAYOUT.map((item, i) => (
				<AnimatedG key={`t-${i}`} animatedProps={props.thinkingAnimatedProps[i]}>
					<Path d={item.iconD} transform="translate(-12 -12)" stroke={colors.mascot?.thinkingIcon ?? '#8A5A36'} strokeWidth={2} strokeLinecap="round" strokeLinejoin="round" fill="none" />
				</AnimatedG>
			))}

			{/* narrating 直上飞出 */}
			{NARRATING_LAYOUT.map((item, i) => (
				<AnimatedG key={`n-${i}`} animatedProps={props.narratingAnimatedProps[i]}>
					<Path d={item.iconD} transform="translate(-12 -12)" fill={colors.mascot?.narratingGlow ?? '#FFC857'} />
				</AnimatedG>
			))}
		</Svg>
	)
}
```

**关键实现细节**：

1. 阴影椭圆 z-order 必须在 `box-body` **之前**（视觉位于盒子下方）
2. 阴影用 G wrapper + `<Ellipse cx=0 cy=0>`，G 的 transform 提供 `translate(100 195) scale(s)`，scale 中心始终是阴影自身中心，不会位移
3. 眼睛 4 path 共享外层 blink G（idle 眨眼时整组 scaleY 压扁）
4. thinking / narrating 元素 G wrapper 在 SVG 树最外层，独立坐标系
5. mascot.svg 中的 `fill="currentColor"` 替换为 `colors.mascot.*` token；新 token 在 `mobile/theme/colors.ts` 补齐并标注 `// M4.7 新增`
6. `<Svg>` 设 `style= overflow: 'visible'` ，确保 enter 时 body translateY=-200 飞入不被裁切

**验证**：

- ⏸ 组件 mount 无报错
- ⏸ 默认 animatedProps 下静态渲染，视觉与 v0.5 原型一致（盒身 + 盒盖 + 脸 + 眼睛 + 阴影）
- ⏸ TypeScript 零报错

```tsx
	// eyes 4 形态 opacity
	const eyesOpenOpacity = useSharedValue(1)
	const eyesSquintOpacity = useSharedValue(0)
	const eyesSmileOpacity = useSharedValue(0)
	const eyesCrescentOpacity = useSharedValue(0)
	// thinking / narrating progress（每元素 0→1 循环）
	const thinkingProgress = THINKING_LAYOUT.map(() => useSharedValue(0))
	const narratingProgress = NARRATING_LAYOUT.map(() => useSharedValue(0))

	useEffect(() => {
		const params = STATE_PARAMS[state]

		// —— 取消所有正在跑的动画 ——
		cancelAnimation(bodyScaleX)
		cancelAnimation(bodyScaleY)
		cancelAnimation(bodyTranslateY)
		cancelAnimation(shadowScale)
		cancelAnimation(shadowOpacity)
		cancelAnimation(blinkScaleY)
		thinkingProgress.forEach(cancelAnimation)
		narratingProgress.forEach(cancelAnimation)

		// —— body 动画 ——
		if (params.isOneTime && state === 'enter') {
			// enter：从屏顶坠落 + 阴影增强 + 落地 squash
			bodyTranslateY.value = -200
			bodyScaleX.value = 1
			bodyScaleY.value = 1
			shadowScale.value = 0.2
			shadowOpacity.value = 0.1

			bodyTranslateY.value = withTiming(0, { duration: 600, easing: EASING.fall })
			shadowScale.value = withTiming(1, { duration: 600, easing: EASING.fall })
			shadowOpacity.value = withTiming(0.5, { duration: 600, easing: EASING.fall })

			bodyScaleX.value = withSequence(
				withDelay(600, withTiming(1.15, { duration: 120, easing: EASING.standard })),
				withTiming(0.92, { duration: 120, easing: EASING.standard }),
				withTiming(1, { duration: 150, easing: EASING.standard }, (finished) => {
					if (finished && onFinish) runOnJS(onFinish)()
				}),
			)
			bodyScaleY.value = withSequence(
				withDelay(600, withTiming(0.85, { duration: 120, easing: EASING.standard })),
				withTiming(1.15, { duration: 120, easing: EASING.standard }),
				withTiming(1, { duration: 150, easing: EASING.standard }),
			)
		} else if (params.isOneTime && state === 'done') {
			// done：轻颔首 scale 1→1.05→1
			shadowScale.value = withTiming(1, { duration: 200 })
			shadowOpacity.value = withTiming(0.3, { duration: 200 })
			bodyTranslateY.value = withTiming(0, { duration: 200 })
			bodyScaleX.value = withSequence(
				withTiming(1.05, { duration: 200, easing: EASING.standard }),
				withTiming(1, { duration: 300, easing: EASING.standard }, (finished) => {
					if (finished && onFinish) runOnJS(onFinish)()
				}),
			)
			bodyScaleY.value = withSequence(
				withTiming(1.05, { duration: 200, easing: EASING.standard }),
				withTiming(1, { duration: 300, easing: EASING.standard }),
			)
		} else if (params.body.loop) {
			// 循环态：scale + translateY 半周期来回
			shadowScale.value = withTiming(1, { duration: 200 })
			shadowOpacity.value = withTiming(0.3, { duration: 200 })

			bodyScaleX.value = withRepeat(
				withSequence(
					withTiming(params.body.scaleX, { duration: params.body.duration, easing: EASING.loopSoft }),
					withTiming(1, { duration: params.body.duration, easing: EASING.loopSoft }),
				), -1, false,
			)
			bodyScaleY.value = withRepeat(
				withSequence(
					withTiming(params.body.scaleY, { duration: params.body.duration, easing: EASING.loopSoft }),
					withTiming(1, { duration: params.body.duration, easing: EASING.loopSoft }),
				), -1, false,
			)
			bodyTranslateY.value = withRepeat(
				withSequence(
					withTiming(params.body.translateY, { duration: params.body.duration, easing: EASING.loopSoft }),
					withTiming(0, { duration: params.body.duration, easing: EASING.loopSoft }),
				), -1, false,
			)
		} else {
			// 非循环、非一次性（如 listen / thinking 静态收紧）：单次 timing 到目标值
			shadowScale.value = withTiming(1, { duration: 200 })
			shadowOpacity.value = withTiming(0.3, { duration: 200 })
			bodyScaleX.value = withTiming(params.body.scaleX, { duration: params.body.duration, easing: EASING.standard })
			bodyScaleY.value = withTiming(params.body.scaleY, { duration: params.body.duration, easing: EASING.standard })
			bodyTranslateY.value = withTiming(params.body.translateY, { duration: params.body.duration, easing: EASING.standard })
		}

		// —— blink ——
		if (params.blinkActive) {
			// 整个周期 4s：scaleY 1→1（3.6s）→ 0.1（100ms）→ 1（100ms）→ 1（200ms padding）
			blinkScaleY.value = withRepeat(
				withSequence(
					withTiming(1, { duration: BLINK_PERIOD_MS - BLINK_DURATION_MS }),
					withTiming(0.1, { duration: BLINK_DURATION_MS / 2 }),
					withTiming(1, { duration: BLINK_DURATION_MS / 2 }),
				), -1, false,
			)
		} else {
			blinkScaleY.value = withTiming(1, { duration: 100 })
		}

		// —— eyes 形态切换（150ms crossfade） ——
		eyesOpenOpacity.value     = withTiming(params.eyes === 'open'     ? 1 : 0, { duration: 150 })
		eyesSquintOpacity.value   = withTiming(params.eyes === 'squint'   ? 1 : 0, { duration: 150 })
		eyesSmileOpacity.value    = withTiming(params.eyes === 'smile'    ? 1 : 0, { duration: 150 })
		eyesCrescentOpacity.value = withTiming(params.eyes === 'crescent' ? 1 : 0, { duration: 150 })

		// —— thinking 抛物线 ——
		if (params.thinkingActive) {
			THINKING_LAYOUT.forEach((item, i) => {
				thinkingProgress[i].value = 0
				thinkingProgress[i].value = withDelay(
					item.delay,
					withRepeat(
						withTiming(1, { duration: item.durationMs, easing: EASING.parabolic }),
						-1, false,
					),
				)
			})
		} else {
			thinkingProgress.forEach(sv => { sv.value = withTiming(0, { duration: 200 }) })
		}

		// —— narrating 直上 ——
		if (params.narratingActive) {
			NARRATING_LAYOUT.forEach((item, i) => {
				narratingProgress[i].value = 0
				narratingProgress[i].value = withDelay(
					item.delay,
					withRepeat(
						withTiming(1, { duration: item.durationMs, easing: EASING.standard }),
						-1, false,
					),
				)
			})
		} else {
			narratingProgress.forEach(sv => { sv.value = withTiming(0, { duration: 200 }) })
		}
	}, [state])

	// —— animatedProps 组装 ——
	const bodyAnimatedProps = useAnimatedProps(() => ({
		transform: `translate(0 ${bodyTranslateY.value}) scale(${bodyScaleX.value} ${bodyScaleY.value})`,
	}))
	const shadowAnimatedProps = useAnimatedProps(() => ({
		opacity: shadowOpacity.value,
		transform: `translate(100 195) scale(${shadowScale.value})`,
	}))
	const blinkAnimatedProps = useAnimatedProps(() => ({
		transform: `scale(1 ${blinkScaleY.value})`,
	}))
	const eyesOpenAnimatedProps     = useAnimatedProps(() => ({ opacity: eyesOpenOpacity.value }))
	const eyesSquintAnimatedProps   = useAnimatedProps(() => ({ opacity: eyesSquintOpacity.value }))
	const eyesSmileAnimatedProps    = useAnimatedProps(() => ({ opacity: eyesSmileOpacity.value }))
	const eyesCrescentAnimatedProps = useAnimatedProps(() => ({ opacity: eyesCrescentOpacity.value }))

	const thinkingAnimatedProps = THINKING_LAYOUT.map((item, i) =>
		useAnimatedProps(() => {
			const p = thinkingProgress[i].value
			const x = item.startX + (item.peakX - item.startX) * p
			const y = item.startY + (item.peakY - item.startY) * p
			const scale = item.scaleStart + (item.scaleEnd - item.scaleStart) * p
			const rotate = item.rotateEnd * p
			// opacity keyframe: 0→1 (p=0..0.2), 1 (0.2..0.7), 1→0 (0.7..1)
			const opacity = p < 0.2 ? p * 5 : p > 0.7 ? Math.max(0, (1 - p) / 0.3) : 1
			return {
				opacity,
				transform: `translate(${x} ${y}) rotate(${rotate}) scale(${scale})`,
			}
		})
	)

	const narratingAnimatedProps = NARRATING_LAYOUT.map((item, i) =>
		useAnimatedProps(() => {
			const p = narratingProgress[i].value
			const y = item.startY + (item.endY - item.startY) * p
			const scale = item.scaleStart + (item.scaleEnd - item.scaleStart) * p
			const opacity = p < 0.15 ? p / 0.15 : p > 0.6 ? Math.max(0, (1 - p) / 0.4) : 1
			return {
				opacity,
				transform: `translate(${item.x} ${y}) scale(${scale})`,
			}
		})
	)

	return (
		<View style={[{ width: px, height: px }, style]}>
			<MascotSvg
				bodyAnimatedProps={bodyAnimatedProps}
				shadowAnimatedProps={shadowAnimatedProps}
				blinkAnimatedProps={blinkAnimatedProps}
				eyesOpenAnimatedProps={eyesOpenAnimatedProps}
				eyesSquintAnimatedProps={eyesSquintAnimatedProps}
				eyesSmileAnimatedProps={eyesSmileAnimatedProps}
				eyesCrescentAnimatedProps={eyesCrescentAnimatedProps}
				thinkingAnimatedProps={thinkingAnimatedProps}
				narratingAnimatedProps={narratingAnimatedProps}
			/>
		</View>
	)
}
```

**关键实现细节（必须严格遵守）**：

1. 状态切换必须先 cancel 再赋值：所有动画启动前 cancelAnimation，避免新旧叠加抖动
2. onFinish 触发条件：只在 `withTiming` callback 内 `if (finished && onFinish) runOnJS(onFinish)()`；只在 enter / done 注册；cancel 场景 finished=false 不触发
3. useEffect 依赖只监听 `state`：size 仅影响外层容器；onFinish 不加依赖（identity 变化避免误重跑）
4. useAnimatedProps 在顶层 hook：`THINKING_LAYOUT` / `NARRATING_LAYOUT` 长度在启动后恒定（icons.ts 初始化即定），.map 等价于 N 次顶层调用，不会触发 hooks 规则警告
5. 阴影 `transform: translate(100 195) scale(s)` 让 G 先平移到阴影中心再缩放，Ellipse 中心是 G 原点 (0,0)，scale 时位置不位移
6. enter 起始 translateY=-200：Mascot.svg.tsx 的 `<Svg>` 已设 `style= overflow: 'visible'` ，飞入不被裁切
7. thinking opacity keyframe：`p<0.2` 渐入、`0.2~0.7` 全显、`p>0.7` 渐出。若效果太硬，调三个区间阈值
8. react-native-svg 的 `transform` 字符串属性可被 useAnimatedProps 驱动；如个别场景未生效，拆为多层 G 嵌套（外层 translate / 中层 rotate / 内层 scale）每层独立 useAnimatedProps

**验证**：

- ⏸ 6 态在展厅可切换

## 五、发现与建议

### 5.1 动效参数表回写

Step 4 STATE_PARAMS 与 Step 5 LAYOUT 当前是首版可运行参数。Iver 在展厅看到效果后，按以下顺序回写本计划：

1. 各态半周期 / scale / translateY 数值微调 → `animations.ts` STATE_PARAMS
2. 眼睛 4 形态 d 字符串按 mascot.svg 实际盒脸坐标微调 → `animations.ts` EYE_*_D
3. thinking / narrating 起终点坐标 + delay + durationMs 微调 → `icons.ts` THINKING_LAYOUT / NARRATING_LAYOUT
4. opacity keyframe 阈值（thinking 0.2/0.7、narrating 0.15/0.6）调硬度 → `Mascot.tsx` useAnimatedProps

回写完成的项打 ✅，未回写的保持当前数值即可继续运行。

### 5.2 Skia 兜底触发条件

本计划不验证 v2 §0 POC 的「FlatList 60fps + 键盘无卡顿」三项指标（M4.5 占位 Mascot 不参与聊天页），延后到 M7 聊天页落地时验证。届时若不达标，再启动 v2 §9 Skia 重写路径。

### 5.3 ClipPath 方案弃用

v2 §3 提出的 `<ClipPath id="cave-visible">` 方案在本计划弃用。原因：

- thinking / narrating 通过 layout 的起终点坐标 + opacity keyframe 实现「从盒口冒出 → 飞出顶部」的视觉，元素飞出 viewBox 顶部即被自然裁切
- ClipPath 需精确量取盒口多边形坐标，对 mascot.svg 实际盒口形状强耦合；首版 4 个 thinking + 1 个 narrating 的稀疏抛洒不需要硬裁切
- 后续若加雾气、密集元素、需要「半截卡在盒口」的视觉效果时，再回头引入 ClipPath

### 5.4 Babel 插件名变更风险

`react-native-worklets/plugin` 是 Reanimated 4 起的新名。旧文档 / AI 生成代码可能仍用 `react-native-reanimated/plugin`，导致 worklet 抛出。Step 1 主动核验，避免走到 Step 6 才发现。

### 5.5 mascot.svg 阴影元素新增

Mascot.svg.tsx 新增的 shadow ellipse 不是 mascot.svg 源文件内容，而是本计划在 Step 3 转换时主动加入。Iver 提供的 `mascot.svg` 可不包含阴影；执行 agent 按 Step 3 代码骨架在 box-body 之前手动添加 ellipse 即可。

### 5.6 enter 坠落起点与 Svg overflow

enter 起始 body translateY=-200（在 viewBox 200×200 的上边界外），依赖 `<Svg style= overflow: 'visible' >` 才能可见。若展厅发现 enter 时盒子是「凭空出现」而非「从顶坠落」，检查该 style 属性是否生效；或者把起始值改为 -100（仅半个 viewBox 高度）避开裁切问题。
